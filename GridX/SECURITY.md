# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

GridX is a research project and does not handle sensitive data. However, if you discover a security issue:

1. **Do not** open a public issue
2. Email the maintainers directly
3. Include detailed reproduction steps

We will respond within 7 days and work with you to address the issue.

## Scope

This policy covers:
- The RTL source code
- Test infrastructure
- Build scripts

Out of scope:
- Third-party tools (iverilog, verilator, sv2v)
- Theoretical vulnerabilities in synthesized hardware
